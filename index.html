Roi Monsa - Automation Project

This project was developed on myTinyToDo platform.
myTinyToDo platform is a user engagement platform that enables you to create tasks,
manage them in different lists, add note for each and priority, edit their properties,
mark them as a Completed tascks or delete them if you wish & more.

The Goal:

Create test cases in order to test myTinyToDo platform's features and functionality.
Build solid automation using POM (Page Object Model), and Java, Selenium, TestNG and Maven tools.


Allure -


To get the best report for the Automation project and to get report readable and clear for anyone I inserted to the project Allure's features and annotations,
And now everyone can anderstand the results of the Automation run.

Code Design

To design the code in the best way I use P.O.M (Page Object Model),
Which give us to writing strong, change-resistant, clean and clear code that anyone can understand.

Methods and Tests

1 / 6
- New Class for every Logical page.
- Split between Methods and Tests.
2 / 6
Base Page
- Selenium operations are wrapped
by methods in this class.
- All Page Objects classes we use
will inherit from this class.
3 / 6
Page Elements
- The Page Elements are defined
once in the project,
which gives us a flexible code.
- All the Elements are defined
on their Logical page class.
4 / 6
Page Methods
- In every Page Objects class
the actions on the Logical page
are realized in Methods.
- In this way we can to call to the
same Method again and again
in the Tests classes.
5 / 6
Page Validation
- For each Page Objects class
there are Validation methods
to check if the tests
Passed or failed.
6 / 6
Tests Classes
- In the Tests classes we just call
to the Method from Page Objects classes
which give us clear and simple tests.
- Each test end with Assert
that call to Validtion Method
from Page Objects classes.
